# architecture
- Build the 10 Platform Core modules as highest priority glue infrastructure: Custom Fields Engine, Global Search, DAM, Approval Workflow, Unified Inbox, Visual Workflow Builder, Public API + Webhooks, No-Code Database, E-Signature, GDPR Data Request Center. Confidence: 0.75
- Enforce three-tier information architecture: (1) Workspace Modules (customer-facing features count toward module stats), (2) Workspace Settings (account/config — separate sidebar section, never in module counts/search), (3) Platform Administration (Super Admin — isolated route/layout/nav with RBAC, never reachable from regular navigation). Confidence: 0.85
- Consolidate fragmented modules into unified workspaces by merging modules sharing the same business domain into a single workspace with tabbed sub-views; preserve every feature, API, permission, integration, and automation during consolidation. Confidence: 0.85
- Hide empty/coming-soon sub-module tabs in consolidated workspaces instead of showing them; only render tabs that have real implementation. Confidence: 0.65
- Every consolidated workspace must include a dashboard, analytics, activity timeline, AI assistant, search, filters, automation, reports, settings, integrations, collaboration, notifications, history, permissions, templates, exports, imports, and audit logs. Confidence: 0.85
- Replace duplicated shared services (file uploads, comments, notes, reminders, notifications, activity logs, AI actions, exports, imports, approvals, attachments, tagging, custom fields) with centralized platform services reusable across all modules. Confidence: 0.85
- Benchmark module architecture against enterprise suites (Salesforce, HubSpot, Zoho, Odoo, ServiceNow) to ensure cohesive workspace organization; use insights to inform grouping but do not copy them directly. Confidence: 0.70
